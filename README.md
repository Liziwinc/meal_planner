
# Дизайн-документ: Конструктор рациона питания (консольное приложение)  
*Версия 2.0 (с выбором трёх блюд на день)*

## 1. Цели и сценарии использования

Приложение позволяет:
- **Сценарий 1:** Добавлять блюда в базу данных: название, **тип (завтрак/обед/ужин)**, список ингредиентов с граммовкой и калориями.
- **Сценарий 2:** Составлять рацион на заданное количество дней, **выбирая для каждого дня по три блюда (завтрак, обед, ужин) из соответствующих списков**, и автоматически формировать список покупок (суммарные ингредиенты) и дневной рацион.

## 2. Функциональные требования

### Основные возможности
1. **Управление блюдами**:
   - Добавление нового блюда с указанием:

     - Названия (уникальное)
     - **Типа (завтрак, обед, ужин)**
     - Списка ингредиентов (название ингредиента, граммовка на порцию, калории на порцию)
   - Просмотр всех блюд (сводный список с краткой информацией)
   - Удаление блюда (опционально, можно добавить позже)
2. **Составление рациона**:
   - Запрос количества дней (1–14, например)
   - **Для каждого дня:**
     - **Вывод списка блюд типа «завтрак» и выбор одного**
     - **Вывод списка блюд типа «обед» и выбор одного**
     - **Вывод списка блюд типа «ужин» и выбор одного**
   - После завершения вывода:
     - **Рацион по дням с указанием трёх приёмов пищи (завтрак, обед, ужин)**
     - **Список покупок: ингредиент → общий вес (граммы) с округлением до 10 г (суммируются ингредиенты всех выбранных 

блюд)**
   - (опционально) Возможность сохранить рацион в файл (txt/csv)

## 3. Структура базы данных

```sql
-- Таблица блюд
CREATE TABLE dishes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE,
    type TEXT CHECK(type IN ('завтрак', 'обед', 'ужин')) NOT NULL
);

-- Таблица ингредиентов (нормализация, чтобы не хранить название ингредиента в каждом блюде)
CREATE TABLE ingredients (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE
);

-- Связь блюда с ингредиентами (количество 

граммов и калории на порцию)
CREATE TABLE dish_ingredients (
    dish_id INTEGER NOT NULL,
    ingredient_id INTEGER NOT NULL,
    grams INTEGER NOT NULL,
    calories_per_portion INTEGER NOT NULL,
    FOREIGN KEY (dish_id) REFERENCES dishes(id) ON DELETE CASCADE,
    FOREIGN KEY (ingredient_id) REFERENCES ingredients(id) ON DELETE CASCADE,
    PRIMARY KEY (dish_id, ingredient_id)
);
```

### Модули и их ответственность

#### `utils.py`
- `get_db_path()` – определяет путь к файлу БД.
- `input_int(prompt, min_val, max_val)` – безопасный ввод целого числа.
- `input_choice(prompt, options)` – выбор из списка вариантов.
- `print_table(headers, rows)` – форматированный вывод таблицы.

#### `database.py`

- `init_db(conn)` – создание таблиц.
- `add_dish(conn, name, dish_type, ingredients)` – вставка блюда и его ингредиентов.
- `get_all_dishes(conn)` – возвращает список блюд (id, name, type).
- **`get_dishes_by_type(conn, dish_type)` – возвращает блюда заданного типа (id, name).**
- `get_dish_details(conn, dish_id)` – возвращает ингредиенты блюда (ingredient_name, grams, calories_per_portion).
- `delete_dish(conn, dish_id)` – удаление блюда (опционально).

#### `main.py`
- Основной цикл с меню:
  1. Добавить блюдо
  2. Показать все блюда
  3. Составить рацион
  4. Удалить блюдо (опционально)
  0. Выход
- Вызов функций из `database.py` и `utils.py`.

## 6. Алгоритмы

### Добавление блюда
1. Запросить название, проверить на уникальность.
2. Запросить тип (числовой выбор: 1-завтрак, 2-обед, 3-ужин).

3. Запросить количество ингредиентов.
4. Для каждого ингредиента:
   - Запросить название, проверить, есть ли в таблице `ingredients`. Если нет – добавить.
   - Запросить граммы и калории.
   - Сохранить в списке.
5. Вставить блюдо в таблицу `dishes`.
6. Для каждого ингредиента вставить запись в `dish_ingredients`.

### Составление рациона
1. Запросить количество дней `N`.
2. **Создать пустой список `selected_dishes = []` (будем хранить id выбранных блюд).**
3. **Для каждого дня от 1 до N:**
   - **Для каждого типа в порядке (завтрак, обед, ужин):**
     - Получить список блюд этого типа из БД (id и название).
     - Если список пуст, вывести сообщение и предложить добавить блюда (или прервать составление).
     - Показать список с номерами.
     - Получить номер выбранного блюда, 

добавить его id в `selected_dishes`.
4. **Построить рацион:** вывести по дням, для каждого дня три блюда с указанием типа.
5. **Построить список покупок:**
   - Создать словарь `shopping_list = {}` (ключ – название ингредиента, значение – общий вес).
   - Для каждого `dish_id` в `selected_dishes`:
     - Получить ингредиенты блюда (название, граммы).
     - Для каждого ингредиента добавить граммы к значению в словаре.
   - Отсортировать словарь по ключам.
   - Вывести ингредиенты и общий вес (округлить до 10 г).

## 8. План сборки через GitHub Actions (мультиплатформенный)

Мы настроим GitHub Actions так, чтобы при каждом push в ветку `main` (или при ручном запуске) создавались исполняемые файлы для трёх основных операционных систем: Windows (`.exe`), Linux (бинарный файл) и 

macOS (бинарный файл).

### Структура workflow

Создайте файл `.github/workflows/build.yml` со следующим содержимым:

```yaml
name: Build executables for Windows, Linux, macOS

on:
  push:
    branches: [ main ]
  workflow_dispatch:    # позволяет запускать вручную

jobs:
  build:
    strategy:
      matrix:
        os: [windows-latest, ubuntu-latest, macos-latest]
        include:

          - os: windows-latest
            artifact_name: meal-planner-windows.exe
            pyinstaller_args: --onefile --console main.py
          - os: ubuntu-latest
            artifact_name: meal-planner-linux
            pyinstaller_args: --onefile --console main.py
          - os: macos-latest
            artifact_name: meal-planner-macos
            pyinstaller_args: --onefile --console main.py

    runs-on: ${{ matrix.os }}

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.10'

      - name: Install dependencies
        run: |
          pip install pyinstaller
          # Если есть requirements.txt, раскомментируйте следующую строку:
          # pip install -r requirements.txt

      - name: Build executable with PyInstaller
        run: |
          pyinstaller ${{ matrix.pyinstaller_args }}

      - name: Upload artifact
        uses: actions/upload-artifact@v4
        with:
          name: ${{ matrix.artifact_name }}
          path: dist/main*   # PyInstaller создаёт файл без расширения на Linux/macOS и .exe на Windows
```

### Проверка на Linux и macOS

- **Linux**: полученный файл `meal-planner-linux` необходимо сделать исполняемым 

(`chmod +x meal-planner-linux`) перед запуском.  
- **macOS**: аналогично, после скачивания возможно потребуется снять блокировку (через `xattr -d com.apple.quarantine meal-planner-macos`), если система не доверяет приложению.
